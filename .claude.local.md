# Claude Local — telegram-bridge

## Key learnings from building this bridge

- `claude --print --output-format stream-json --verbose` — streaming JSON output format, requires `--verbose`
- `claude.cmd` (Windows batch) works in Python subprocess; `claude` (POSIX shell script) does not on Windows
- Stream output: parse `type=assistant` → `message.content[].text` and `type=result` → `result`; deduplicate by `message.id`
- Windows stderr may be GBK-encoded; use `decode("gbk", errors="replace")`
- `asyncio.create_subprocess_shell` on Windows needs `claude.cmd` with full path in a string command
- `asyncio.wait()` on both stdout+stderr to avoid deadlock from full buffer
- Session IDs are process-bound; cannot be shared across independent `claude --print` invocations
- `--no-session-persistence` with `--print` gives cleaner output
- `--session-id` must be valid UUID format
- JSON log file needs file locking for concurrent writes; currently skipped (assume single-writer)
